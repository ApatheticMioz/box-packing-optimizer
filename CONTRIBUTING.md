# Contributing to Box Packing Optimizer

Thank you for considering contributing to this project! This document outlines the guidelines for contributing.

## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in the Issues section.
2. If not, create a new issue with a clear title and description.
3. Include steps to reproduce the bug and expected vs. actual behavior.

### Suggesting Features

1. Open an issue describing the feature and its use case.
2. Explain why this feature would be beneficial.

### Pull Requests

1. Fork the repository.
2. Create a new branch for your feature or bugfix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes and ensure they follow the existing code style.
4. Test your changes thoroughly.
5. Commit your changes with a clear message:
   ```bash
   git commit -m "Add: description of your changes"
   ```
6. Push to your fork and submit a pull request.

## Code Style

- Follow PEP 8 style guidelines for Python code.
- Use 4 spaces for indentation.
- Include comments for complex logic.
- Keep functions focused and modular.

## Development Setup

1. Clone the repository:
   ```bash
   git clone https://github.com/ApatheticMioz/mt1008-project-a.git
   cd mt1008-project-a
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Run the application:
   ```bash
   python main.py
   ```

## Questions?

Feel free to open an issue for any questions or concerns.
